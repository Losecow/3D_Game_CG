## [2:00 – 5:00] WebGL · 컴퓨터 그래픽스 소개

**A:** 근데 이게 어떻게 브라우저에서 3D로 보이는 걸까요?

비밀은 **WebGL**이에요. 웹 브라우저가 GPU에 직접 접근해서 3D 그래픽을 그리는 기술인데요 — 앱 설치 없이 링크만 있으면 어디서든 3D 게임을 실행할 수 있어요.

**B:** 쉽게 말하면, 여러분 폰에서도 이 링크 하나면 지금 바로 이 게임을 할 수 있다는 거예요.

*(모바일 접속 보여주기 또는 QR 코드 표시)*

**A:** 3D 물체를 그리는 건 어떻게 하냐면 — 수박 하나가 화면에 보이기까지 이런 과정이 있어요.
점들을 연결해서 구 모양을 만들고 → 거기에 텍스처(이미지)를 입히고 → 조명을 계산해서 빛이 반사되는 걸 표현해요.
그 계산을 GPU가 초당 60번 하고 있어요.

*(과일을 카메라로 회전시키며 보여주기)*

**B:** 이걸 처음부터 다 짜면 엄청난 양인데 — 여기서 AI가 어떤 역할을 했는지가 오늘 핵심이에요.

## [8:30 – 14:00] 핵심 — LLM 처리 과정 시각화

**A:** 지금부터 그 과정을 직접 보여드릴게요.

*(🔬 버튼 클릭 → ON 상태로)*

이 버튼 켜면, 다음 명령부터는 AI가 어떻게 처리하는지 단계별로 보여줘요.

> 입력: `딸기 오른쪽에 놓아줘`

*(오버레이 등장 — 4단계 애니메이션)*

**B:** 자, 화면에 뭔가 떴죠? 같이 읽어볼게요.

---

### Step 1 — 입력 분석

**B:** 첫 번째 단계예요. AI가 문장을 받아서 중요한 단어들을 찾아내요.

*(칩들이 등장하는 모습 보면서)*

"딸기" — 과일 키워드. "오른쪽에" — 위치 키워드. "놓아줘" — 동작 키워드.

**A:** 우리가 문장을 읽을 때 무의식적으로 하는 게 있잖아요 — "이 문장에서 중요한 게 뭐지?" 하고요. AI도 그걸 하는 거예요.

---

### Step 2 — 프롬프트 구성

**A:** 두 번째 단계. 보라색 태그 보이세요? "시스템 지시"예요. 저희가 AI한테 미리 써준 설명서예요.

읽어보면 — *"너는 과일 합체 3D 게임의 AI 컨트롤러다. 자연어 명령 → JSON 변환"* 이렇게 적혀 있어요.

**B:** 쉽게 말하면 AI한테 "너는 이 게임 담당이야, 이런 명령들이 있어" 하고 역할을 정해준 거예요. 그리고 파란색 태그가 방금 우리가 입력한 거고요.

**A:** 이 두 개가 합쳐진 게 실제로 AI한테 전달되는 내용이에요.

---

### Step 3 — Gemini API 호출

**B:** 세 번째. 지금 "전송 중..." 이라고 깜빡이죠?

이 순간 우리 서버가 구글 Gemini AI한테 요청을 보내고 있어요. 인터넷 너머 구글 AI 서버로요.

**A:** *(응답 도착 후 JSON 타이핑되는 모습 보며)*

응답이 왔어요. 이런 형태예요.

```json
{"action": "drop_fruit", "params": {"level": 1, "x_ratio": 0.8}}
```

**B:** JSON이라는 데이터 형식인데요 — "동작은 drop_fruit, 과일 레벨은 1번(딸기), 위치는 오른쪽(0.8)"이라고 숫자로 정리된 거예요. 자연어를 컴퓨터가 이해하는 언어로 바꾼 거죠.

---

### Step 4 — 해석 & 실행

**A:** 마지막 단계. 이 JSON을 게임 코드가 받아서 실제로 과일을 떨어뜨렸어요.

*(화면에 딸기가 오른쪽에 드롭된 모습 확인)*

**B:** 정리하면 — 우리가 한국어로 말했고, AI가 이해해서 JSON으로 변환했고, 게임이 실행했어요. 이 네 단계가 방금 몇 초 안에 일어난 거예요.

## [17:00 – 19:00] "저희도 AI랑 같이 만들었어요"

**B:** 사실 고백 하나 하면요 — 이 게임 자체도 AI랑 같이 만들었어요.

**A:** 예를 들어 방금 여러분이 본 저 LLM 처리 과정 오버레이요 — 이걸 AI한테 이렇게 말했어요.

> *"처리 과정을 새 화면에 띄워서 보여주고 싶어. 문장이 어떻게 나뉘고 분석되고 응답이 오는지 단계별로 애니메이션으로."*

그랬더니 AI가 HTML, CSS, JavaScript 코드를 써줬고, 저희가 검토해서 적용했어요.

**B:** AI가 코드를 다 짜줬냐고요? — 완전히는 아니에요. AI가 초안을 쓰면 저희가 틀린 부분 고치고, 원하는 대로 조정하고, 다시 AI한테 물어보고. 그 과정을 반복하는 거예요.

**A:** 비유하면 AI는 엄청 빠른 조수예요. "이거 만들어봐" 하면 5초 안에 초안을 줘요. 근데 그게 항상 맞는 건 아니라서, 뭐가 좋고 뭐가 틀린지 판단하는 건 결국 사람이 해야 해요.
